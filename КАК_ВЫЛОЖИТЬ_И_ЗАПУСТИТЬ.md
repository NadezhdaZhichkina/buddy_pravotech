# Как выложить Buddy на GitHub и запустить в интернете

## Часть 1: Выложить на GitHub

### Шаг 1.1 — Создать токен на GitHub

1. Открой https://github.com/settings/tokens
2. Нажми **Generate new token** → **Generate new token (classic)**
3. Название: `buddy-deploy`
4. Срок: 90 дней (или No expiration)
5. Отметь галочку **repo**
6. Нажми **Generate token**
7. **Скопируй токен** (он показывается один раз, сохрани его)

### Шаг 1.2 — Выложить код в терминале

Открой **Терминал** и выполни команды по очереди:

```bash
cd /Users/nadezhda/buddy_2.0
```

```bash
git add .
git status
```

(Должны появиться файлы для коммита. Если пусто — всё уже закоммичено.)

```bash
git commit -m "Buddy: база знаний + деплой"
```

```bash
git remote add pravotech https://github.com/NadezhdaZhichkina/buddy_pravotech.git
```

Если напишет `remote pravotech already exists` — тогда:
```bash
git remote set-url pravotech https://github.com/NadezhdaZhichkina/buddy_pravotech.git
```

```bash
git push pravotech main
```

Когда спросит:
- **Username:** `NadezhdaZhichkina`
- **Password:** вставь **токен** из шага 1.1 (не пароль от GitHub!)

## Часть 2: Запустить в интернете (Streamlit Cloud)

### Шаг 2.1 — Зайти на Streamlit Cloud

1. Открой https://share.streamlit.io
2. Нажми **Sign up** и войди через **GitHub**
3. Разреши доступ к своим репозиториям

### Шаг 2.2 — Создать приложение

1. Нажми **New app**
2. **Repository:** выбери `NadezhdaZhichkina/buddy_pravotech`
3. **Branch:** `main`
4. **Main file path:** `streamlit_app.py`
5. Нажми **Deploy**

Подожди 1–2 минуты. Появится ссылка вида:
`https://buddy-pravotech-xxx.streamlit.app`

### Шаг 2.3 — Добавить секреты (чтобы всё работало)

1. В приложении на share.streamlit.io нажми **Settings** → **Secrets**
2. Вставь (замени значения на свои):

```toml
OPENROUTER_API_KEY = "sk-or-v1-твой-ключ"
OPENROUTER_MODEL = "openai/gpt-4.1-mini"
```

**Если есть PostgreSQL** (Supabase, Neon и т.п.) — добавь:
```toml
STREAMLIT_DATABASE_URL = "postgresql://user:password@host:6543/dbname"
```

> **Supabase:** используй порт **6543** (pooler), а не 5432. Порт 5432 — прямое подключение, на Streamlit Cloud может не работать.

**Без PostgreSQL** — Buddy будет работать, но база знаний сохранится в файл (на Streamlit Cloud это временно, лучше добавить БД).

3. Нажми **Save**

Приложение перезапустится с новыми настройками.

### Шаг 2.4 — Устранение неполадок

**«Received no response from server» (1ST)** — приложение не отвечает при первом запуске:

1. **Проверь подключение к БД.** Если указан `STREAMLIT_DATABASE_URL` (PostgreSQL), убедись, что:
   - для Supabase используется порт **6543** (pooler), не 5432;
   - хост, логин и пароль верны.
2. **Попробуй SQLite.** Временно убери `STREAMLIT_DATABASE_URL` из Secrets — Buddy переключится на локальный SQLite и должен запуститься. Если заработает, проблема в PostgreSQL.
